About me

01 Profile

"Everything I'm not made me everything I am."

— Kanye West

I'm a second-year Computer Science and Mathematics student at the University of Alberta. I got into this field to create and to learn. Simple as that.

To me, coding is like a complex puzzle: super tricky until you learn how to solve it, and the way you learn is practice, repetition, and creation. I learn programming in class, but that only takes me so far. Everything that's actually stuck came from building: a quant platform trained on 122,000+ rows of market data, a neural network written in raw NumPy, no frameworks, only math. Right now I'm exploring autonomous agents and distributed systems, building things that are out of my level, then learning along the way to close the gap. Everything I couldn't do yet is exactly what taught me the most.

Outside of coding I'm usually playing basketball, reading, or watching anime. Recently I found out I enjoy building stuff based on the things I like, for example an app based around basketball. I believe if you focus on what you like instead of what bores you, you tap into your creative side, and that's what makes you productive.

MB.aiMB.ai

2026 — In development

An autonomous agent that completes multi-step tasks on its own

  • Building a tool system — web search, sandboxed code execution, file operations, calculator — each exposed to the LLM as a callable function.
  • An agent loop where the model picks a tool, reads the result, decides the next step, and repeats until the task is done — with memory carried across steps.
  • Sandboxed execution so the agent writes and runs code without destroying anything, plus a frontend to hand it a task and watch it work step by step.

Mini RedisMini Redis

2026 — In development

A distributed key-value store, written from first principles

  • Starts as a single-node TCP server handling GET/SET/DELETE — no database libraries doing the heavy lifting.
  • Expands to multi-node replication with Raft consensus: leader election, log replication, and writes confirmed only once replicated.
  • Built for fault tolerance — kill any node and the cluster keeps working — with a CLI client to drive it. End goal: slot it under Vestr as the cache layer.
